# SPN UX iteration: document role breakdown

Дата: 27 июня 2026.

## Что изменено

В `assets/js/nav-v2/spn-next-actions-v2.js` добавлен разбор обязательных документов по ответственным ролям.

Теперь СПН видит:

- общий разбор по всем обязательным документам к контролю;
- отдельный разбор по ролям внутри блокера задатка или сделки.

Пример для текущей тестовой сделки:

- все обязательные документы: `СПН: 4`, `юрист: 7`;
- блокер задатка: `юрист: 7`.

## Подключение

`deal-card-v2.html` подключает модуль как:

`spn-next-actions-v2.js?v=20260627-0200`

## Цель

СПН должен быстро понять, кого подключать к устранению блокера, без чтения всех документов по одному.

## Область изменений

Изменён только интерфейс Навигатора сделок v2.

CRM «Лидер» не затрагивалась.
