# Таймаут карточки сделки

Дата: 25 июня 2026.

## Проверка

Проверено: проблема не в правах и не в размере карточки.

Серверная RPC карточки отвечает быстро, карточка небольшая.

## Исправления

1. Общий RPC timeout в `supabase-v2.js` поднят до 45 секунд.

2. В карточку подключен аварийный модуль:

`assets/js/nav-v2/deal-card-timeout-recovery-v2.js`

Он ловит ошибку таймаута, делает повторную загрузку с обходом кэша и при повторной ошибке показывает минимальную карточку через прямой fallback-запрос.

3. Добавлена страница проверки:

`deal-card-check-v2.html`

Она использует отдельный модуль:

`assets/js/nav-v2/deal-card-check-v2.js`

и импортирует `supabase-v2.js` с явной версией.

## Безопасность

Закрыт прямой execute для helper-функции:

`nav_v2_handoff_gap_count(uuid)`

Обычные пользователи получают handoff score только через batch RPC с проверкой видимости сделки.

Миграция сохранена:

`supabase/migrations/20260625114000_navigator_revoke_direct_handoff_gap_count.sql`

## Ограничение

Полный cache-bust внутреннего импорта `supabase-v2.js` внутри большого `deal-card-v2.js` пока не выполнен. Для этого нужно отдельное безопасное обновление большого файла.

CRM «Лидер» не затрагивалась.
