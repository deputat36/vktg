# Статус: СПН UX, сценарий объекта Доля

Дата: 2026-06-23 15:25, уточнено 15:35 и 15:45
Профиль проверки: Овчинников Александр Константинович, a.k.ovchinnikov@borisoglebsk.etagi.com, роль СПН

## Проблема

В мастере СПН есть два уровня описания объекта:

- физический тип недвижимости: квартира, комната, дом с участком, земля, новостройка, коммерция;
- юридическая форма: целый объект или доля / часть объекта.

Серверная функция `nav_v2_save_wizard_result` не принимала legacy-значение `objectType = share`. Это могло ломать старые черновики или быстрые сохранения, где доля попадала в payload как тип объекта, а не как юридическая форма. Для СПН это риск неверной классификации сделки, ошибок в маршрутизации к юристу и путаницы при обучении новичков.

## Что изменено

- В БД `nav_v2_save_wizard_result` теперь принимает `share` как допустимый legacy `objectType`.
- Миграция добавлена в репозиторий: `supabase/migrations/20260623152500_navigator_allow_share_object_in_spn_wizard.sql`.
- В `spn-v2.html` оставлен правильный пользовательский маршрут: сначала физический объект, затем блок `Юридическая форма объекта` с выбором `Целый объект` или `Доля / часть объекта`.
- Старая базовая кнопка `data-action="set:objectCategory:share"` скрыта, чтобы она не мигала до перестройки интегрированного шага и не сбивала СПН.
- `spn-object-step-integrated-v2.js` теперь при нормализации старого черновика с `objectType = share` сохраняет физический тип из `shareBaseObject`, если он есть.
- Версии ассетов мастера подняты до `?v=20260623-1545`, чтобы браузер забрал актуальный код.

## Проверка

- Под JWT Овчинникова выполнено транзакционное сохранение legacy-сделки с `objectType = share`.
- Под JWT Овчинникова выполнено транзакционное сохранение нормального маршрута: `objectType = flat_mkd`, `legalForm = share`, `shareSale = true`, `shareBaseObject = flat_mkd`, `flags = [shares]`.
- В обоих случаях функция вернула draft-карточку с `risk_level = yellow` и следующим действием `Передать юристу основания права и риски`.
- Транзакции завершены `rollback`.
- Контрольные запросы по тестовым id и адресам вернули `0` строк, тестовые сделки в базе не остались.

## Эффект для СПН

СПН выбирает понятный физический объект, затем отмечает, что продается доля / часть объекта. Это снижает риск неверной классификации и сохраняет нужный юридический маршрут. Старые payload с `objectType = share` теперь не ломают сохранение, а старые черновики с `shareBaseObject` не теряют выбранный физический объект.

## Границы изменения

CRM `Лидер` не затрагивалась. Изменения ограничены `nav_v2_*`, мастером СПН v2 и документацией итерации.
