# Статус: СПН UX, сценарий объекта Доля

Дата: 2026-06-23 15:25
Профиль проверки: Овчинников Александр Константинович, a.k.ovchinnikov@borisoglebsk.etagi.com, роль СПН

## Проблема

В мастере СПН был сценарий `Доля`, но страница `spn-v2.html` скрывала кнопку через inline CSS, а серверная функция `nav_v2_save_wizard_result` не принимала `objectType = share`. Для СПН это означало неочевидный обходной путь: выбирать другой тип объекта и описывать долю в комментарии. Это повышало риск ошибок в документах, маршрутизации к юристу и обучении новичков.

## Что изменено

- В `spn-v2.html` удалено скрытие `data-action="set:objectCategory:share"`.
- Версии ассетов мастера подняты до `?v=20260623-1525`, чтобы браузер забрал актуальный код.
- В БД `nav_v2_save_wizard_result` теперь принимает `share` как допустимый `objectType`.
- Миграция добавлена в репозиторий: `supabase/migrations/20260623152500_navigator_allow_share_object_in_spn_wizard.sql`.

## Проверка

- Под JWT Овчинникова выполнено транзакционное сохранение сделки с `objectType = share`.
- Функция вернула draft-карточку с `risk_level = yellow` и следующим действием `Передать юристу основания права и риски`.
- Транзакция завершена `rollback`.
- Контрольный запрос по тестовому id и адресу вернул `0` строк, тестовая сделка в базе не осталась.

## Эффект для СПН

СПН теперь может явно выбрать объект `Доля`, получить правильный юридический маршрут и не прятать критичный тип объекта в комментариях. Для новичков это снижает риск неверной классификации сделки.

## Границы изменения

CRM `Лидер` не затрагивалась. Изменения ограничены `nav_v2_*`, мастером СПН v2 и документацией итерации.
