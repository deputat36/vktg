# Navigator v2 — быстрый consultation intake

Дата: 16 июля 2026 года.

## Статус

`repository-only preview`

Экран `consultation-v2.html` не сохраняет консультацию в Supabase, не создаёт сделку, документы, риски или задачи и не добавлен в production role menu.

## Зачем нужен отдельный вход

Текущий полный мастер рассчитан на подготовку задатка или сделки. Для обычного юридического вопроса он слишком длинный и способен раньше времени создать большой document/task backlog.

Целевой короткий сценарий:

`вопрос СПН → минимальные факты → понятная передача юристу → ответ / уточнение / преобразование в подготовку`

## Один экран

Preview собирает только:

- конкретный вопрос или требуемый результат;
- сторону сопровождения;
- стадию;
- тип объекта;
- безопасный ориентир без точного помещения;
- источник средств;
- особые обстоятельства;
- плановую дату;
- безопасные известные факты;
- HTTPS-ссылку на утверждённый внешний источник документов.

ФИО, телефоны, email, паспортные данные, СНИЛС, номера карт, кадастровые номера и номера квартир/комнат/офисов запрещены.

## Роли

Preview доступен после проверки профиля только ролям:

- СПН;
- юрист;
- менеджер;
- owner;
- admin.

Ипотечный брокер и viewer в юридический consultation intake не допускаются.

## Ипотечный брокер

Ипотечный брокер подключается параллельно только при ипотеке или военной ипотеке и отвечает за:

- ипотечную консультацию;
- подбор программы;
- одобрение банка;
- обучение СПН ипотечным сценариям.

Маткапитал и сертификат без ипотеки остаются в контуре СПН и юриста.

При сочетании ипотеки с маткапиталом или сертификатом ипотечный брокер ведёт только ипотечную часть, а СПН и юрист — правовую и расчётную схему и оформление сделки.

## Результат preview

Экран формирует готовый текст передачи для eChat и показывает предварительный маршрут.

Автоматический маршрут не является юридическим заключением.

Пользователь может:

1. скопировать текст передачи;
2. перенести уже введённые безопасные факты в локальный draft полного мастера;
3. продолжить подготовку без повторного ввода этих фактов.

Существующий draft полного мастера имеет приоритет и не перезаписывается импортом консультации.

## Будущий серверный lifecycle

После отдельного review требуется repository-only серверный prototype:

- consultation entity с минимальным DTO;
- статусы `new`, `need_info`, `answered`, `converted`, `closed`;
- решения юриста `answer`, `need_info`, `convert_to_preparation`;
- очередь юриста без клиентских идентификаторов;
- преобразование в подготовку только по явному действию;
- отсутствие полного document/task backlog до `convert_to_preparation`;
- audit events для ответа, уточнения и преобразования.

## Проверки

Синтетическая матрица проверяет:

- обычный юридический вопрос;
- срочный вопрос до задатка;
- ипотеку;
- военную ипотеку;
- маткапитал без ипотеки;
- сертификат без ипотеки;
- ипотеку вместе с маткапиталом;
- телефон в свободном тексте;
- точный номер помещения;
- кадастровый номер;
- возможные ФИО;
- небезопасную HTTP-ссылку.

## Production gate

До добавления официального пункта меню и серверного сохранения необходимы:

1. repository-only consultation schema/RPC prototype;
2. privacy allowlist DTO;
3. synthetic lifecycle scenarios;
4. authenticated role/mutation E2E;
5. подтверждённый rollback;
6. отдельный PR и review deployment.

## Rollback

Repository rollback состоит из удаления:

- `consultation-v2.html`;
- `assets/css/nav-v2-consultation.css`;
- `assets/js/nav-v2/consultation-intake-model-v2.js`;
- `assets/js/nav-v2/consultation-v2.js`;
- consultation fixtures/checkers/workflow;
- записи страницы из module budget.

Supabase rollback не требуется: production база, функции, grants, RLS, Auth и Edge Functions не изменяются.
