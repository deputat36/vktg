# Статус итерации СПН UX — 2026-06-24 08:37 CEST

## Цель

Исключить открытие существующей чужой карточки после тайм-аута сохранения новой сделки.

## Выявленный риск

Резервный поиск после тайм-аута сначала искал карточку по адресу, а затем мог принять любую доступную сделку того же типа объекта. Для распространённых типов недвижимости это небезопасная корреляция.

## Изменения

- RPC-клиент запоминает ID сделок из последнего успешно загруженного списка.
- Перед вызовом `nav_v2_save_wizard_result` фиксируется снимок уже существующих ID.
- Если сохранение завершилось успешно, режим восстановления сразу сбрасывается.
- Если сохранение завершилось ошибкой или тайм-аутом, последующие проверочные запросы списка возвращают только ID, которых не было в снимке.
- Старая сделка с совпадающим типом объекта больше не попадает в резервный поиск мастера.
- Если снимок списка получить не удалось, проверочный список становится пустым: мастер не открывает непроверенную карточку автоматически.
- Режим восстановления ограничен двумя минутами и завершается после обнаружения новой записи.
- В `spn-v2.html` версия входного модуля мастера обновлена до `20260624-0815`.
- Параллельное обновление версии проверки дублей `20260624-0740` сохранено без отката.
