# Статус итерации СПН UX — 2026-06-24 09:44 CEST

## Цель

Исключить ложные предупреждения об устаревшей сделке, когда команда работала с задачами, документами, комментариями или историей карточки.

## Выявленная проблема

Индикатор актуальности использовал только `nav_deals_v2.updated_at`. Изменения связанных задач, документов и комментариев сохраняются в отдельных таблицах и не обязаны менять эту дату.

## Изменения

- В `nav_v2_get_deals_list` добавлено поле `last_activity_at`.
- Последняя активность вычисляется как максимум из:
  - изменения основной записи сделки;
  - изменения задач;
  - изменения документов;
  - добавления комментариев;
  - событий истории сделки.
- Существующее поле `updated_at`, порядок выдачи, лимит и правила видимости не менялись.
- Интерфейс СПН использует `last_activity_at` и сохраняет резервный переход на `updated_at` для обратной совместимости.
- Текст индикатора теперь говорит об активности в карточке, а не только об изменении основной сделки.
- Версия модуля обновлена до `20260624-0930`.
- Миграция сохранена: `supabase/migrations/20260624074219_nav_v2_deals_list_last_activity.sql`.
