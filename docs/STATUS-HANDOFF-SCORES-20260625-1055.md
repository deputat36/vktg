# Handoff scores для списка сделок

Дата: 25 июня 2026.

## Что сделано

Добавлен серверный слой оценки готовности передачи сделки юристу:

- `nav_v2_handoff_gap_count(uuid)` — считает количество базовых пробелов передачи.
- `nav_v2_get_handoff_scores(jsonb)` — одним запросом возвращает score по списку сделок.

Batch-RPC нужна для списка сделок, чтобы не делать отдельный запрос по каждой карточке.

## Что проверяется

- ФИО продавца.
- Телефон продавца.
- ФИО покупателя.
- Телефон покупателя.
- Тип объекта.
- Адрес объекта.
- Цена.
- Задаток.
- Расчёты.
- Расходы.
- Обязательные незакрытые документы.
- Открытые задачи СПН.

## Интерфейс

Обновлен модуль:

`assets/js/nav-v2/deals-handoff-summary-v2.js`

Теперь он использует серверную RPC `nav_v2_get_handoff_scores`, а локальный расчёт оставлен как fallback.

В `deals-v2.html` модуль подключен напрямую с версией:

`deals-handoff-summary-v2.js?v=20260625-1050`

## Проверка

Проверено под СПН Овчинниковым:

- `nav_v2_get_handoff_scores(...)` вернула 4 элемента по доступным сделкам.
- В каждом элементе есть `handoff_gap_count`.
- В каждом элементе есть `handoff_readiness_score`.
- В каждом элементе есть `handoff_ready`.

Права:

- `authenticated` — execute есть.
- `service_role` — execute есть.
- `anon` — execute не выдан.

## Ограничение

Supabase-миграции были успешно применены в базе.

GitHub connector заблокировал сохранение SQL-файлов миграций для helper/batch RPC. Поэтому текущее состояние базы рабочее, но эти две последние SQL-миграции нужно будет сохранить в репозиторий позже, когда connector пропустит запись или при ручном commit.

CRM «Лидер» не затрагивалась.
