# Статус итерации СПН UX — 2026-06-23 21:57 CEST

## Цель

Не допустить, чтобы ошибка загрузки списка сделок воспринималась как успешная проверка отсутствия дублей.

## Проверка реализации

- Активный мастер сохраняет сделку кнопкой `data-action="save"`.
- Резервный вариант мастера использует кнопку `#saveDealBtn`.
- Защитные модули подключены до основного мастера и перехватывают событие сохранения на capture-фазе.

## Изменения

- Проверка дублей теперь различает состояния «совпадений нет» и «список сделок не загрузился».
- При недоступной автоматической проверке СПН видит постоянное предупреждение рядом со статусом страницы.
- При попытке сохранения требуется отдельное подтверждение продолжения без проверки дублей.
- Подтверждение привязано к текущему адресу и типу объекта и действует 30 секунд.
- После ошибки загрузки прежнее подтверждение сбрасывается.
- Проверка дублей и проверка обязательных полей распознают обе финальные кнопки: активную и резервную.
- Версии защитных модулей в `spn-v2.html` обновлены до `20260623-2215`.
