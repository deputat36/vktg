## Зачем

Backend уже возвращает для каждой задачи поле `can_change_status`. Основной интерфейс частично проверяет права локально, поэтому добавлен дополнительный слой защиты на frontend: кнопки задач принудительно сверяются с серверным флагом.

Это нужно, чтобы СПН не видел и не нажимал действия, которые backend все равно отклонит.

## Поведение

`task-action-guard-v2.js`:

- загружает карточку через `nav_v2_get_deal_card(...)`;
- собирает карту задач по `task.id`;
- проверяет `task.can_change_status`;
- отключает недоступные кнопки `button[data-task-id][data-task-status]`;
- визуально приглушает недоступные кнопки;
- добавляет пояснение: статус задачи меняет ответственный специалист;
- перечитывает права после действия с задачей.
