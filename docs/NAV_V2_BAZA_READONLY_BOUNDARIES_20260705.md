# Navigator v2 + BAZA read-only boundaries — 2026-07-05

## Принцип

Navigator v2 — рабочее пространство сделки. BAZA — источник знаний. Интеграция не должна смешивать эти роли.

## Первый этап

В карточке сделки можно показывать короткие подсказки из BAZA по типу сделки, этапу, роли пользователя и рискам. Подсказки должны быть только для чтения.

## Нельзя передавать в BAZA

- ФИО клиента.
- Телефон клиента.
- Полный адрес объекта.
- Паспортные, банковские и договорные данные.
- Персональные комментарии сотрудников.

## Можно использовать как сигнал

- Тип объекта.
- Этап сделки.
- Наличие ипотеки.
- Наличие маткапитала или сертификата.
- Тип юридического риска без персональных деталей.
- Статус документов.
- Роль пользователя: СПН, юрист, брокер, менеджер.

## UI в карточке сделки

Блок: `Подсказки по сделке`.

Показывать 3–5 коротких подсказок:

- название;
- для какой роли;
- почему показано;
- ссылка на материал, если он есть.

## Границы

- Без записи из Navigator в BAZA.
- Без автоматического обучения на реальных сделках.
- Без изменения `leader_*` и `parket_*`.
- Без блокировки релиза PR #154.

## Критерий готовности

- Есть утверждённый формат подсказки.
- Есть список сигналов сделки.
- Есть UI-макет блока подсказок.
- Есть решение: хранить approved knowledge в приватных `kb_*` таблицах или брать из отдельного read-only endpoint.
