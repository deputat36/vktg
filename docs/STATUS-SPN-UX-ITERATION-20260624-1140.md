# СПН UX: автоматические сроки задач

Дата: 2026-06-24
Профиль проверки: Овчинников Александр Константинович (`a.k.ovchinnikov@borisoglebsk.etagi.com`), роль `spn`.

## Проблема

Автоматические задачи мастера создавались без `due_date`. Из-за этого список сделок сообщал СПН о задачах без срока, а после каждой новой сделки даты приходилось назначать вручную.

## Реализовано

- Добавлен триггер `nav_deal_tasks_v2_auto_due_date` для задач с источником `auto_%`.
- Срочные задачи получают срок `current_date + 1`.
- Важные задачи получают срок `current_date + 2`.
- Обычные задачи получают срок `current_date + 5`.
- Остальные автоматические задачи получают срок `current_date + 7`.
- Существующим открытым автоматическим задачам без даты назначены актуальные будущие сроки.
- Для массового назначения созданы события `task_due_date_initialized` с `actor_id = null` и причиной `automatic_default`.
- RPC изменения срока теперь отклоняет дату в прошлом.
- В карточке сделки срок виден также на задачах юриста и брокера, но СПН не может его менять.
- Поле даты в интерфейсе ограничено текущим днем и будущими датами.

## Проверка

- Миграции Supabase:
  - `20260624091859_nav_v2_auto_task_due_dates.sql`;
  - `20260624091930_nav_v2_validate_task_due_date.sql`.
- Все 40 открытых автоматических задач получили сроки; задач без срока осталось 0.
- Срочные задачи получили дату 25 июня 2026 года, важные — 26 июня 2026 года.
- Тестовая важная автозадача получила `current_date + 2`; транзакция откатилась.
- Попытка Овчинникова установить срок в прошлом отклонена; транзакция откатилась.
- В трех доступных Овчинникову сделках `overdue_tasks_count = 0`, ближайший срок — 25 июня 2026 года.
