# Статус: СПН UX, точная задача юристу по доле

Дата: 2026-06-23 16:15
Профиль проверки: Овчинников Александр Константинович, a.k.ovchinnikov@borisoglebsk.etagi.com, роль СПН

## Проблема

После добавления документов и риска по доле сделка получала общий следующий шаг `Передать юристу основания права и риски` и общую задачу `Юридическая проверка до задатка`. Для СПН и юриста этого недостаточно: предмет проверки доли нужно видеть сразу, без чтения всех рисков и документов.

## Что изменено

В `nav_v2_save_wizard_result` для долевых сделок (`flags` содержит `shares` или legacy `objectType = share`):

- `next_action` теперь становится конкретным: `Передать юристу долю: уведомления, отказы и порядок пользования`;
- создается отдельная urgent-задача юристу:
  - source: `auto_share_lawyer`;
  - title: `Проверить долю / часть объекта`;
  - description: проверить уведомления/отказы сособственников, порядок пользования, фактическое пользование, конфликт и нотариальный сценарий.

Миграция в репозитории: `supabase/migrations/20260623161500_navigator_add_explicit_share_lawyer_task_to_spn_wizard.sql`.

## Проверка

Под JWT Овчинникова выполнено транзакционное создание долевой сделки:

- `objectType = house_land`;
- `legalForm = share`;
- `shareSale = true`;
- `shareBaseObject = house_land`;
- `flags = [shares]`.

Результат внутри rollback-транзакции:

- `next_action = Передать юристу долю: уведомления, отказы и порядок пользования`;
- созданы две urgent-задачи юристу: общая юридическая проверка и точная задача `auto_share_lawyer`;
- после `rollback` контрольный запрос по тестовому id и адресу вернул `0` строк.

## Эффект для СПН

СПН видит конкретный следующий шаг по доле, а юрист получает отдельную задачу без необходимости вручную вычитывать смысл из общего риска. Это ускоряет передачу и снижает вероятность пропустить преимущественное право покупки или порядок пользования.

## Границы изменения

CRM `Лидер` не затрагивалась. Изменение ограничено `nav_v2_save_wizard_result`, миграцией и документацией итерации.
