# Статус: СПН UX, автоматический чек-лист для доли

Дата: 2026-06-23 16:05
Профиль проверки: Овчинников Александр Константинович, a.k.ovchinnikov@borisoglebsk.etagi.com, роль СПН

## Проблема

После выбора сценария `Доля / часть объекта` мастер СПН корректно отправлял сделку к юристу, но автоматический список документов оставался базовым. СПН и новичок могли не увидеть обязательные вопросы по доле: преимущественное право покупки, уведомления/отказы сособственников, порядок пользования и фактическое пользование.

## Что изменено

В `nav_v2_save_wizard_result` добавлен блок для долевых сделок (`flags` содержит `shares` или legacy `objectType = share`):

- создаются специальные документы категории `share`:
  - `Размер доли и выписка ЕГРН по доле`;
  - `Уведомления сособственников или отказы`;
  - `Соглашение или решение о порядке пользования`;
  - `Информация о фактическом пользовании долей`;
- создается отдельный риск `Продажа доли / части объекта` с назначением на юриста;
- риск блокирует задаток и сделку до юридической проверки.

Миграция в репозитории: `supabase/migrations/20260623160500_navigator_add_share_documents_and_risk_to_spn_wizard.sql`.

## Проверка

Под JWT Овчинникова выполнено транзакционное создание сделки:

- `objectType = flat_mkd`;
- `legalForm = share`;
- `shareSale = true`;
- `shareBaseObject = flat_mkd`;
- `flags = [shares]`.

Результат внутри rollback-транзакции:

- создано 4 документа категории `share`;
- общий список документов вырос до 12;
- создан риск категории `share` с `assigned_role = lawyer`;
- после `rollback` контрольный запрос по тестовому id и адресу вернул `0` строк.

## Эффект для СПН

СПН сразу получает понятный чек-лист по доле, не держит критичные вопросы в голове и не передает юристу неполную заявку. Для новичков это снижает риск пропустить уведомления сособственников или порядок пользования до задатка.

## Границы изменения

CRM `Лидер` не затрагивалась. Изменение ограничено `nav_v2_save_wizard_result`, миграцией и документацией итерации.
