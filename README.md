# VKTG — Telegram-пайплайн «Парсер → Модерация → Публикация»

Готовый к запуску сервис, который читает посты из заданных источников, отправляет их на модерацию и по подтверждению публикует в целевой канал без дублей. Все сообщения и логи — на русском языке.

## Возможности

- Поддержка одиночных сообщений и альбомов (фото, видео, документы).
- Очередь модерации в приватном чате с кнопками: `Approve`, `Reject`, `Edit caption`, `Retry`, `Open source`.
- Правки текста перед публикацией и опциональная строка атрибуции.
- Устойчивость к перезапускам: состояние хранится в SQLite, медиа складываются в файловое хранилище.
- Автоматические повторы при ошибках публикации, журнал статусов и команды `/ping`, `/status`.
- Профили `local` и `cloud` переключаются одной настройкой `PROFILE`.

## Структура

```
src/
  config.py            — загрузка настроек из YAML
  logging_config.py    — настройка логов с ротацией
  storage.py           — SQLite-хранилище очереди
  media.py             — описание полезной нагрузки
  source_watcher.py    — подписка на источники через Telethon
  moderation_bot.py    — бот модерации (python-telegram-bot)
  publisher.py         — публикация одобренных материалов
  main.py              — точка входа и координация компонентов
config.example.yaml    — шаблон настроек
.env.example           — шаблон секретов
requirements.txt       — зависимости Python
run.bat / stop.bat     — запуск и остановка сервиса в Windows
```

## Подготовка окружения (Windows 10/11)

1. Установите [Python 3.11+](https://www.python.org/downloads/windows/) с галочкой «Add Python to PATH».
2. Распакуйте проект в удобную папку, например `C:\vktg`.
3. Скопируйте `config.example.yaml` → `config.yaml` и `.env.example` → `.env`.
4. Откройте `config.yaml`, заполните параметры `SOURCES`, `TARGET`, `MOD_CHAT`, при необходимости включите `ATTRIBUTION_ENABLED`, настройте `DEDUP_TTL_DAYS` и профиль `cloud`.
5. В `.env` пропишите `BOT_TOKEN`, `API_ID`, `API_HASH`, `SESSION_NAME` (опционально).
6. Запустите `run.bat`. Скрипт автоматически создаст виртуальное окружение `.venv`, установит зависимости и поднимет сервис в отдельном окне.
7. В модераторском чате отправьте `/ping` — бот ответит «Я на связи ✅». Команда `/status` выдаёт текущие счётчики.

Логи записываются в каталог `logs` (можно изменить в конфиге). Ротация работает автоматически.

## Корректная остановка

Используйте `stop.bat` — скрипт найдёт PID сервиса из `service.pid`, отправит сигнал завершения и дождётся остановки. Дополнительных действий не требуется.

## Профиль `cloud`

В `config.yaml` доступна секция `profiles.cloud`. Достаточно установить `PROFILE: cloud`, чтобы сервис использовал альтернативные пути хранилища и специфичные параметры (например, каталог в подключённом облачном диске). Поведение пайплайна идентично локальному. Возврат к `PROFILE: local` не приводит к дублям — ключи дедупликации и очередь лежат в выбранном профилем хранилище. Логи по-прежнему доступны в каталоге, указанном в `LOGGING.DIRECTORY` (можно настроить путь в облаке).

## Проверка работы

1. Добавьте бота и/или пользователя Telethon в источники и модераторский чат с правами чтения.
2. Опубликуйте тестовый пост в одном из источников. В течение ~60 секунд появится карточка в модераторском чате.
3. Нажмите `Approve` — пост попадёт в целевой канал в исходном формате (альбом сохранится целиком). `Reject` исключает пост из очереди навсегда.
4. Кнопка `Edit caption` позволяет задать новую подпись. `Retry` доступен, если публикация завершилась ошибкой.

## Диагностика

- Файл `logs/service.log` содержит подробные сообщения с таймстемпами.
- При сетевых ошибках публикатор выполняет ретраи с экспоненциальной паузой (настраивается в конфиге).
- В случае критической ошибки публикации элемент меняет статус на `failed`, а в карточке появляется блок «Ошибка публикации» и активируется `Retry`.

## Зависимости

Устанавливаются автоматически из `requirements.txt`:

- `python-telegram-bot` — модераторский бот и публикация.
- `telethon` — чтение каналов-источников от имени пользователя/бота.
- `python-dotenv` — загрузка секретов из `.env`.
- `PyYAML` — парсинг конфигурации.

## Скрипты Windows

### `run.bat`

- Создаёт (при необходимости) виртуальное окружение `.venv` рядом с проектом.
- Устанавливает зависимости.
- Запускает `python -m src.main` в отдельном окне «VKTG Service».
- PID процесса сохраняется в `service.pid`.

### `stop.bat`

- Читает PID из `service.pid`.
- Посылает `CTRL_BREAK_EVENT` и `taskkill` при необходимости.
- Удаляет `service.pid` после успешной остановки.

## Примечания

- Секреты не хранятся в репозитории — используйте `.env`.
- При первом запуске Telethon создаст файл сессии (`SESSION_NAME.session`) в рабочем каталоге.
- Для публикации альбомов требуется, чтобы бот имел права на отправку медиа и группы медиа в целевом канале.
