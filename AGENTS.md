# AGENTS.md

## Проект

Репозиторий CRM «Навигатор сделок»: `deputat36/vktg`.
Основная ветка: `main`.

Перед работой прочитать `README.md`, `docs/PROJECT.md`, `docs/STATUS.md` и `docs/DECISIONS.md`.

## Границы данных

Общая авторизация с CRM «Лидер» допустима. Прикладные данные «Навигатора сделок» должны находиться только в сущностях с префиксом `nav_`.

Не использовать таблицы CRM «Лидер» для сделок, клиентов, заказов, каталога, ролей и профилей Навигатора.

## Правила изменений

- Изменения базы оформлять миграциями.
- Проверять RLS и права затронутых ролей.
- Не выполнять опасные операции с рабочими данными без отдельного подтверждения владельца.
- Не сохранять закрытые учетные данные в репозитории.
- Использовать отдельную ветку.
- Вносить минимально необходимые изменения.
- Не создавать дублирующие сущности без необходимости.
- Не ломать роли СПН, юриста, ипотечного брокера, менеджера, администратора и owner.
- После задачи обновлять `docs/STATUS.md`.
- Важные архитектурные решения добавлять в `docs/DECISIONS.md`.
- Перед PR выполнять доступные проверки.
